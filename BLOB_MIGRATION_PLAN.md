# Blob 图片存储迁移计划

## 目标

把浏览器数据库中的图片正文从 Data URL 字符串切换为 Blob。新的导入数据直接写入 Blob，旧 Data URL 数据通过“迁移为 Blob 存储”批量转换。

## 为什么要改

- Data URL 会把二进制图片变成更大的字符串，图库变大后更容易占内存。
- Blob 更适合 IndexedDB 存大图片，导入导出时也更接近原始文件数据。
- Blob URL 只在页面需要显示图片时临时生成，离开视野后释放，避免长期占住内存。

## 本次改动范围

1. 新增统一图片数据层：
   - 读取图片字节：同时支持 Blob 与旧 Data URL。
   - 显示图片：按需生成 Blob URL。
   - 释放图片 URL：卡片重绘、切换模型、全屏关闭、页面卸载时回收。
2. 导入路径改为写入 Blob：
   - 图片导入。
   - 向已有卡片盒追加图片。
   - 本地压缩包/文件夹读取。
3. 去重与导出改为读取统一图片字节：
   - 哈希优先使用已保存的 hash。
   - 没有 hash 时从图片字节计算。
   - 导出压缩包时直接写入图片字节。
4. 底部预览缩略图：
   - 新导入图片会自动生成 `thumbBlob`。
   - 缩略图最长边为 160px，只用于卡片盒底部小预览。
   - 上方大图和全屏预览仍使用原图 Blob。
   - 导出压缩包时只导出原图，不导出缩略图缓存。
   - 旧数据可通过“检查缩略图”批量补齐。
5. 提供一次性迁移入口：
   - 位置：数据管理 -> 兼容性维护 -> 迁移为 Blob 存储。
   - 点击后先扫描旧 Data URL 图片数量，再确认迁移。
   - 迁移会保留图片正文、文件名、提示词、工作流、哈希等信息。
6. 提供缩略图检查入口：
   - 位置：数据管理 -> 兼容性维护 -> 检查缩略图。
   - 点击后先扫描缺失缩略图数量，再确认生成。

## URL 缓存释放策略

- 当前卡片渲染时，为卡片主图生成原图 URL，为底部小图优先生成缩略图 URL。
- 卡片重新渲染、离开虚拟列表、切换模型或刷新列表时释放对应 URL。
- 全屏预览使用独立 URL，关闭或切换图片时释放旧 URL。
- 页面关闭前统一释放全部 Blob URL。
